Requested as: QST-046

## Summary

## Implementation Notes

- Added `tags` and `entity_tags` tables.
- Tags are owner-scoped and normalized for search/deduplication.
- Entity tags support `quest`, `mission`, `trail`, and `arc_memory`.
- Quest, Mission, Trail, and Arc Memory persistence now trigger best-effort tagging.
- Tagging failures do not block core save flows.
- `TaggingService` exposes:
  - `tagQuest`
  - `tagMission`
  - `tagTrail`
  - `tagArcMemory`
  - `searchByTag`
  - `stats`
- Local deterministic tag generation is available now and can be swapped for an Edge Function later without changing callers.
