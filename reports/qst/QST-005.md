## Implementation summary

- Added storage-key mapping for `TrailType` so app values map cleanly to the
  existing Supabase `trails.trail_type` check constraint.
- Added a `TrailRepository` interface with in-memory and Supabase
  implementations.
- Added `trailRepositoryProvider`, matching the existing Supabase-configured
  repository pattern used by Arc Memory.
- Connected `TrailController` to load trails for the signed-in user and persist
  newly created Quest/Mission Trail records through the repository boundary.
- Reused the existing `public.trails` schema and RLS policies already present in
  `supabase/migrations/202606130001_mvp_schema.sql`.

## Known issues

- Trail persistence runs in the background and does not yet expose loading,
  saved, or failed states in the UI.
- Local mock Trail records remain as seed data when no remote user trails are
  loaded.

## Next QST candidates

- Connect Arc Chat to persisted journey context.
- Add Trail sync states and error handling.
- Add Trail editing and deletion through the repository boundary.

## Master Spec compliance notes

- Trail now has a clear adapter boundary for Supabase persistence.
- The implementation remains local-first when Supabase environment values are
  not configured.
