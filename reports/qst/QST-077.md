# QST-077 Report: Trail Timeline V1

## Summary

Added a Trail Timeline view so users can revisit Quest, Mission, Reflection, and manual Trail entries in chronological groups. The existing Trail list remains available, while the new Timeline gives a more memory-like journey view.

- apps/mobile/lib/features/trail/trail_timeline_service.dart

## Implementation Notes

- Added deterministic grouping of Trails by day, newest first.
- Added Timeline UI with date groups, source type pills, reflection marker, Arc marker, and media attachment marker.
- Kept the existing Trail list and edit/media/reflection actions unchanged.
- Empty Timeline state uses Arc guidance.
- Visible labels use Trail terminology.

## Acceptance Check

- Trail Timeline renders recent Trails in chronological order.
- Empty state uses Arc guidance.
- Timeline handles media and Reflection Trails without crashing.
- Existing Trail list behavior remains available.
- No Story terminology was introduced.
