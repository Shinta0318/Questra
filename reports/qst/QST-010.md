## Implementation summary

- Wired `QuestController` to load remote Quests when an authenticated profile is
  available.
- Persisted Quest add/update actions through `QuestRepository`.
- Added delete persistence with optimistic local removal and rollback on remote
  failure.
- Kept user-visible sync state for a later QST to avoid widening this change.
