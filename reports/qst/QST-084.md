# QST-084 Arc Animation Event Layer

Status: Done
Date: 2026-06-21

## Goal

Add a renderer-neutral animation event interface for Arc reactions so the current PNG renderer and future Rive/GLB renderers can share the same reaction vocabulary.

## Implementation

- Added `ArcAnimationEvent`, `ArcAnimationEventType`, and `ArcAnimationIntensity`.
- Added centralized presets for idle, blink, celebrate, concern, thinking, and transition events.
- Added an emotion-to-animation resolver for current Arc emotion states.
- Updated `ArcWidget` to consume `ArcAnimationEvent` metadata for pulse, tilt, duration, and looping.
- Added `ArcExpressionDecision.animationEvent` so expression decisions include both visual asset and motion intent.
- Kept the current PNG renderer and fallback behavior unchanged.

## Acceptance

- Animation events are centralized and testable.
- `ArcWidget` can consume basic events without visual regression.
- No new heavy animation engine is required.
- Future renderer support is documented through renderer hints.
