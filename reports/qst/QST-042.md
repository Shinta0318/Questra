# QST-042 Trail media delete and replace management

## Status

Done

## Summary

Added lifecycle management for private Trail image attachments. Trail cards now
surface the current private image metadata and let users remove or replace the
attached image from the Trail action menu.

## Changes

- Extended `MediaRepository` with Trail image lookup, replace, and delete APIs.
- Added Supabase media deletion that removes the private storage object and
  media row.
- Added replacement cleanup so a failed old-image delete attempts to remove the
  newly uploaded replacement.
- Added Trail media attachment state loading and updates in `TrailController`.
- Updated Trail cards to show attached image metadata.
- Added `Replace image` and `Remove image` actions for attached Trail images.
- Updated Trail media readiness documentation.
- Marked QST-042 as Done in the backlog.

## Acceptance

- Users can see when a Trail has an attached private image.
- Users can replace an attached Trail image.
- Users can remove an attached Trail image.
- Trail media sync success and failure continue to use the Trail sync banner.
