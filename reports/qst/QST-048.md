# QST-048 Report: Arc Presence System

## Summary

Arc presence has been standardized across the MVP surfaces with a reusable `ArcPresence` widget. Home, Arc Chat, Mission, Trail, and Quest now use the same expression-resolution flow or sizing rules for Arc guidance moments.

## Changes

- Added `ArcPresence` and `ArcPresenceSurface` for consistent Arc sizing, card padding, speech, and optional actions.
- Updated Home to render the welcome card through `ArcPresence`.
- Updated Arc Chat to render the main guidance card through `ArcPresence`.
- Updated Mission to use the expression engine for the top guidance card and Arc-led empty state.
- Updated Trail to add Arc guidance and Arc-led empty state.
- Updated Quest hero to consume `ArcExpressionEngine` output instead of a hard-coded Arc expression.
- Added widget tests for Arc presence sizing and message rendering.

## Acceptance

- Arc appears with consistent sizing and spacing across Home, Arc Chat, Quest, Mission, Trail, and empty states.
- Presence rules distinguish top-level guide surfaces and empty-state surfaces.
- Expression behavior flows through `ArcExpressionEngine` where journey context is available.
- Arc remains framed as a navigator and companion.
