# QST-065 Report: AI Quest Guide Generation

Requested as: QST-043

## Summary

Implemented the Arc Guide generation flow for newly created Quests. Quest creation now starts an Arc Guide request, Quest Detail displays the generated guide with thinking/fallback states, and generated Mission candidates can be adopted into saved Missions.

## Implementation Notes

- Quest creation starts Arc Guide generation and then navigates to Quest Detail.
- `ArcQuestGuideService` uses a Supabase Edge Function when configured and falls back to deterministic local generation.
- The generated guide includes:
  - Quest summary
  - Completion path
  - Cautions
  - Arc encouragement
  - Three or more Mission candidates
- Quest Detail shows Arc thinking UI while generation is running.
- Failure keeps the screen usable and offers manual Guide generation again.
- Adopting a Mission candidate uses the existing Mission persistence path, including Supabase save and Arc Memory extraction where a profile is available.
- Generated guide history is recorded through Arc Memory with `questUpdated` source metadata when a profile is available.

## Acceptance Check

- Arc Guide is displayed after Quest registration via Quest Detail generation state.
- Three or more Mission candidates are generated by the local fallback and Edge Function fallback.
- Mission candidates can be adopted from Quest Detail.
- Adopted Missions use the existing DB save path.
- Guide generation history is sent to Arc Memory when user profile context exists.
- Arc is presented as a navigator/companion, not with prohibited product framing.
