# QST-103 Safe Area and Overflow Fix

## Status

Done

## Summary

Added repeatable compact-width and keyboard-inset rendering checks for the
major MVP screens, then fixed every RenderFlex overflow reproduced by those
checks.

## Test Conditions

- Viewport: 320 x 700 logical pixels.
- Text scale: 1.6.
- Keyboard inset checks: 300 logical pixels.
- Screens: Home, Quest, Mission, Trail, Guild, Arc Chat, Profile, and
  onboarding.

## Issues Found And Fixed

1. Home captain status, Stardust, and rank metrics overflowed by 129 pixels.
   The fixed row now wraps across lines when space is limited.
2. Trail Timeline metadata overflowed by 11 pixels after scrolling on a compact
   viewport. Time, type, Arc, media, and highlight indicators now wrap.
3. Trail card type, date, and action menu overflowed by 15 pixels. Header
   metadata now wraps while keeping date and menu grouped.

## Keyboard And Safe Area Verification

- Arc Chat input remains above a simulated keyboard inset.
- Trail create sheet remains scrollable with the keyboard visible.
- Major screens continue to use `SafeArea` and vertical scrolling.
