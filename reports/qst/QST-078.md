# QST-078 Report: Trail Highlight System

## Summary

Added deterministic Trail highlighting so meaningful Trails can become Star Memory candidates. Highlights are based on Reflection depth, Quest/Mission linkage, media presence, and written detail, then displayed inside Trail Timeline.

## Changed Files

- apps/mobile/lib/features/trail/trail_highlight_service.dart
- apps/mobile/lib/features/trail/trail_screen.dart
- apps/mobile/lib/features/trail/trail_timeline_widget.dart
- apps/mobile/test/trail_highlight_service_test.dart
- docs/qst/BACKLOG.yaml
- docs/qst/FEATURE_EXPANSION_BACKLOG.yaml
- docs/product/qst_backlog.md

## Implementation Notes

- Added `TrailHighlightService` and `TrailHighlight`.
- Low-signal Trails are filtered out instead of over-promoted.
- Star Memory candidate threshold is deterministic and test-covered.
- Trail Timeline now shows highlight reasons and a Star Memory candidate marker.
- No backend or paid feature dependency was added.

## Acceptance Check

- Trail highlights are deterministic and testable.
- Highlight reason is visible to the user.
- Highlighted Trails can be used as Star Memory candidates.
- Low-signal Trails are not over-promoted.
- Timeline remains stable for media and Reflection Trails.
