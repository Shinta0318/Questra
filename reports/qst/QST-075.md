# QST-075 Report: Quest Milestone System

## Summary

Added a Quest Milestone foundation so each Quest can be understood as smaller progress checkpoints instead of only a flat percentage. Quest Detail now shows generated or persisted Milestones, lets users save a generated plan, and supports status updates without changing the existing Quest core model.

## Implementation Notes

- Added `QuestMilestone` with status, progress, sort order, and optional Guide type.
- Added in-memory and Supabase repository boundaries.
- Added deterministic milestone planning from Quest, Guide, and Mission context.
- Added Quest Detail Milestones section with save and status update actions.
- Added Supabase table, indexes, and RLS policies for Quest-owned milestones.
- Kept the implementation incremental and avoided changing existing Quest/Mission architecture.

## Acceptance Check

- Quest Detail shows Milestones with status and progress.
- Milestone status can be updated from the UI.
- Generated milestone plans can be saved through the repository boundary.
- Mission context influences milestone progress.
- Supabase migration and RLS policies are present.
- No Story terminology or assistant framing was introduced.
