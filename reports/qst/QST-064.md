# QST-064 Report: Arc AI Chat Integration

## Summary

Arc Chat now uses a service boundary that can call a Supabase Edge Function backed by OpenAI when configured, while preserving a local Arc-native fallback for development and failure cases. The chat flow includes Quest, Mission, Trail, Reflection, and Arc Memory context, shows a thinking state while waiting, and stores completed exchanges as Arc Memory when a profile is available.

## Changes

- Added `ArcChatService`, `LocalArcChatService`, and `SupabaseArcChatService`.
- Added `ArcChatContext`, `ArcChatMessage`, and `ArcChatResponse`.
- Added `arcChatServiceProvider`.
- Rebuilt `ArcScreen` as a `ConsumerStatefulWidget` with:
  - user input handling
  - response waiting state
  - Arc thinking bubble
  - quick action prompts
  - contextual service calls
  - fallback response handling
  - Arc Memory persistence for chat exchanges
- Added `supabase/functions/arc-chat/index.ts`.
  - Uses `OPENAI_API_KEY` and optional `OPENAI_MODEL` when available.
  - Falls back to deterministic Arc-native response when OpenAI is unavailable.
- Added `arc_chat_service_test.dart`.

## Context Included

- Active Quest title, description, progress, and category.
- Recent Mission title and status.
- Recent Trail title, summary, type, and links.
- Important visible Arc Memory title/content/type/importance.
- Recent chat history.

## Acceptance

- Arc Chat can return generated contextual responses via the `arc-chat` function.
- Active Quest context is included in the request.
- Recent Mission and Trail context is included in the request.
- Arc Memory context is included in the request.
- Thinking UI appears while waiting.
- Failure falls back to an Arc-native message.
- User input and Arc response can be saved as Arc Memory.
- Arc remains framed as a star navigator.
