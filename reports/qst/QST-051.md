# QST-051 Report: Questra MVP Performance and Asset Optimization

## Summary

Questra MVP performance limits are now explicit in code, list reads are bounded, Supabase payloads are reduced with column selections, Trail image uploads are pre-sized, and repeatable performance checks are documented.

## Changes

- Added `QuestraPerformanceLimits` for list limits, Arc asset budget, Arc display size, Trail image dimensions, and image quality.
- Added Supabase list limits and explicit column selections for Quest, Mission, Trail, Quest Guide, Trail Event, media, and Arc Memory reads.
- Limited Arc Memory visible reads to important and recent memories.
- Limited Trail image metadata reads to one attachment by default.
- Added Trail image picker max dimensions and quality before upload.
- Removed Home and Guild build-time sorting in favor of repository/controller ordering and small display slices.
- Connected `ArcWidget` default display size to the shared Arc asset display limit.
- Added README performance check steps for profile builds and beta readiness.
- Added Arc PNG size budget coverage in tests.

All Arc PNG assets are below the MVP budget of 307,200 bytes.

## Performance Targets

- Home first render: 1.5 seconds or less.
- Quest and Trail list render: 1 second or less.
- Route transition: 300 ms or less.
- Arc Chat pending state should show waiting or thinking UI.
- Scrolling target: stable 60 fps.

## Acceptance

- Arc PNG display and size limits are defined.
- Trail image picking applies max dimensions and compression quality before upload.
- Quest, Mission, Trail, Quest Guide, Trail Event, media, and Arc Memory reads have limits.
- Arc Memory reads are constrained by visibility, importance, and recency.
- Supabase reads avoid broad `select()` calls in optimized repositories.
- Performance verification steps are documented in `README.md`.
