# QST-057 Report: Stardust Foundation

## Summary

Introduced Stardust as a lightweight progress resource tied to meaningful MVP actions. Stardust is visible in Profile and grows from Quest, Mission, Trail, Reflection, and Arc conversation events.

## Implementation Notes

- Added `StardustService`.
- Stardust awards:
  - Quest created: +10
  - Mission created: +3
  - Mission completed: +8
  - Trail posted: +6
  - Trail reflection: +10
  - Arc conversation: +1
- Quest updates intentionally do not award Stardust.
- `AuthController.addStardust` updates local profile state and persists `stardust_balance` when Supabase is configured.
- Profile now displays Stardust balance and explanatory copy.

## Acceptance Check

- Stardust state has a clear owner through `UserProfile.stardustBalance`.
- Stardust is visible in Profile.
- Stardust is awarded from approved MVP actions.
- Stardust does not require payments or store mechanics.
