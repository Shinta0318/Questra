# QST-041 RLS behavior test harness

## Status

Done

## Summary

Added a database-backed RLS behavior test harness for MVP private data
boundaries. The harness seeds repeatable owner and other-user records inside a
transaction, switches authenticated JWT context, verifies expected visibility and
blocked writes, then rolls back all seed data.

## Changes

- Added `supabase/tests/rls_behavior.sql`.
- Added `tools/qst/run_rls_behavior_tests.ps1`.
- Documented the database-backed RLS test command in `tools/qst/README.md`.
- Marked QST-041 as Done in the backlog.

## Coverage

- Quest: owner private read, other private denial, public read, blocked
  other-owned insert.
- Mission: owner read, other denial through private Quest, public Quest Mission
  read, blocked insert on another user's private Quest.
- Trail: owner private read, other private denial, public read.
- Arc Memory: strictly owner-private reads and blocked other-user insert.
- Media metadata: owner private read, other private denial, public read, blocked
  other-owned insert.

## Verification

- Passed: `dart run tools\qst\verify_rls_readiness.dart`

## Notes

The SQL behavior test requires a Supabase/Postgres database with the project
migrations applied and an authenticated role available. It is intentionally not
part of the default Flutter test run so local UI-only development remains fast.
